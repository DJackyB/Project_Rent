# 包租婆 v0.6 执行清单（按当前策划文本）

## 1) 单回合流程

### 1.1 准备阶段
- [x] 先结算场上卡牌【前置效果】（含合同）
- [x] 再进行抽卡
  - [x] 首回合与非首回合抽卡数可配置

### 1.2 行动阶段
- [x] 打出符合条件的手牌
  - [x] 扣除金钱（花费）
  - [x] 结算【即时效果】
- [x] 点击按钮结束回合进入结算阶段

### 1.3 结束流程（结算阶段）
- [x] 收租/结算
  - [x] 遍历房间，仅当房间存在租客时，结算该房间卡牌【结算效果】
  - [x] 该房间结算后，对房间内卡牌做【耐久】-1
  - [x] 耐久<=0触发销毁流程（默认触发【销毁效果】）
- [x] 合同结算
  - [x] 遍历所有合同卡，处理其【结算效果】与耐久结算
- [x] 弃牌/等待
  - [x] 遍历手牌做【等待】-1
  - [x] 等待归零：租客卡直接移除不触发销毁；非租客卡入弃牌堆
- [x] 扣款判定
  - [x] 每隔 X 回合执行扣款
  - [x] 扣款金额按指数增长（base * growth^n）
- [x] 新牌入库
  - [x] 若游戏继续，执行“3选1”入手牌（当前为无 UI 的自动选取实现）

## 2) 系统一致性
- [x] 金钱仍为唯一资源，行动依赖金钱
- [x] GameOver 使用统一状态信号（`TurnManager.IsGameOver`）
- [x] GameOver 后不再推进回合流程
- [x] 动态新增房间时索引与列表下标一致

## 4) 与当前项目代码对比（2026-03-18）

### 4.1 对比结论
- [x] 准备阶段实际顺序与清单一致：先结算场上【前置效果】（含合同），再按首回合/非首回合规则抽卡。
- [x] 行动阶段与清单一致：出牌时校验费用、扣钱并结算【即时效果】；支持玩家手动结束行动阶段。
- [x] 结算阶段主流程与清单一致：房间结算（仅有租客房间）→ 房间耐久 → 合同结算/耐久 → 等待倒计时 → 扣款判定 → 新牌入库（三选一自动选取）。
- [x] 扣款机制与清单一致：按间隔回合触发，金额按 `base * growth^n` 递增；扣款失败后设置统一 GameOver 状态并停止后续流程。
- [x] 手牌等待归零规则与清单一致：租客卡直接移除不触发销毁，非租客进入弃牌堆。
